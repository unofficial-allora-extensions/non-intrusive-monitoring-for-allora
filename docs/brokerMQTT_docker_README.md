#brokerMQTT_docker README
The configurations included in this folder are meant to be used with the official Grafana/Mosquitto MQTT/Telegraf/InfluxDB docker images, to build the so-called "TIG Stack", executed in a Raspberry Pi or any kind of PC. This software stack represents the data ingestion (MQTT+Telegraf), storage (InfluxDB) and presentation (Grafana) of the monitorization layer of the project. The UI provided by Grafana (with an example directly importable from the "grafana_dashboard_62d92158" JSON in this directory) is meant to be used by the network administrators of the deployments, so they can monitorize the metrics generated by the sensor nodes, and send the control commands of their choice.

Depending of the location of this MQTT broker, the "mqtt_host" parameter in the initialization of the "Requester.py" class must be configured (with the correspondig IP address)
