## Electrical control scheme — LilyGO T3S3 end node

The electrical control scheme used for the LilyGO T3S3 end node is based on a simple 4×AA battery holder, with a MOSFET Power Module acting as a switch. The LilyGO board provides a battery connector whose positive terminal shares the left terminal of the MOSFET module with the positive terminal of the battery holder, while its negative terminal is connected to the central terminal of the MOSFET module, labeled LOAD. This connection represents the load to be switched, so that the power supply is interrupted by disconnecting its ground path. Finally, the right terminal of the MOSFET module receives the negative terminal of the battery holder.

Because its operation is based on disconnecting the return path to ground, this electronic switch is a low-side switch. It works by switching the negative pole between the one used by the load and the one used by the power source. To do so, it receives a control signal through the contact labeled PWM on the MOSFET module. A high logic level (1) on this signal closes the current return path for the load terminal, thus powering the LilyGO. Conversely, a low logic level (0) opens that circuit, causing the controlled node to power off.

It should also be noted that the module itself is powered through the same PWM line, together with a ground reference that must be shared with both the controller node and the controlled node, i.e. the common ground used in their serial I2C interconnection. Since the control signal is provided by another LilyGO board, the PWM terminal is connected to one of its GPIO pins. Several of these pins are available for this purpose; in this implementation, IO38 was used.

Finally, for the default state of the MOSFET to keep the LilyGO ground return closed — that is, for the controlled node to remain powered on — the control signal must normally be held at a high level, except when a reset command is issued by the controller. This is achieved by configuring the startup file of the controller LilyGO nodes (boot.py) so that, upon boot, pin IO38 is set to 3.3 V with respect to ground, which is sufficient to drive the PWM control input.

## Electrical control scheme — Raspberry Pi 3 end node

In the Raspberry Pi 3 case, the final circuit uses a direct connection to the power supply (the common electrical infraestructure) and more robust switching hardware. The Raspberry Pi 3 receives its ground from the power adapter, which is also used as the common ground for the rest of the circuit, including other devices and the reference used in the UART serial interconnection. The positive 5 V supply also comes from the adapter and is first connected to the COM terminal of an electromagnetic relay. When no control signal is applied to the relay — that is, when the relay is in its resting state — this voltage reaches the Raspberry Pi 5V power pin through the normally closed (NC) terminal. The normally open (NO) terminal is not used in this design, so the Raspberry Pi remains powered by default.

The relay control section is arranged as follows. The VCC pin, which powers the relay module itself, is connected to the same positive output of the power adapter that feeds the controlled load. The GND pin is connected to the common ground, as in the rest of the circuit. Finally, the IN pin acts as the relay control input. This pin is connected to the drain terminal of a transistor, whose gate is connected to GPIO 38 of the controller LilyGO node, and whose source is connected to the common ground.

In this way, when the controller sets a low logic level (0) on that GPIO pin, the transistor is not activated. As a result, the relay module’s own internal pull-up resistor keeps the IN pin at a stable 5 V, leaving the relay in its resting state. Under these conditions, the COM–NC connection remains closed and the Raspberry Pi stays powered on. On the other hand, when the controller sets a high logic level (1) on its control pin, the transistor is activated and pulls the relay IN pin down to ground. Consequently, the relay activates, the COM–NO connection closes, the COM–NC connection opens, and the Raspberry Pi power supply is cut.

Beyond the different operating principle imposed by the use of different hardware, note that in this case the positive supply line is switched, rather than the negative one. Since the devices in the circuit may possess multiple ground references -multiple connections to the same GND-, this approach is more stable and effective in a more demanding scenario such as powering a Raspberry Pi. In addition, the transistor stage is necessary for the relay to be controlled by the LilyGO node: if the GPIO pin were connected directly to the relay IN pin, it would not be possible to drive the relay correctly. While maintaining the low logic level required to keep the Raspberry Pi powered would be possible, the relay module requires a 5 V high logic level for activation, whereas LilyGO GPIO pins can only provide 3.3 V for a high logic level. As a result, the Raspberry Pi 3 power control logic is effectively the inverse of the one used in the LilyGO scheme: setting a low logic level on the control pin keeps the Raspberry Pi powered, while setting a high logic level cuts its power supply.
